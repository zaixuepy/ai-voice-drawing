# VoiceDraw 项目 — 2026年6月12日 进度总结

## 今日改进总览（产品经理视角）

### 一、核心体验：从"文字转语音"到"真正的对话"

**改进前**：用户点击麦克风 → 说一句话 → 等待 AI 返回 → 手动点击选项 → 再点麦克风 → 重复。本质是"把键盘输入换成了语音输入"。

**改进后**：用户点一次麦克风 → 持续对话。AI 播报 → 自动开麦 → 用户直接说话打断 → AI 理解并执行。全程无需手操作。

**关键改动**：
- 语音识别器改为持续活跃模式（不再每次停止后重新启动）
- TTS 播报完毕后自动进入监听状态
- 用户开口说话时立即打断 AI 播报
- Clarify 选项支持语音选择（说"一/二/三/四"或"A/B/C/D"即可）

### 二、可靠性：JSON 解析从脆弱到鲁棒

**改进前**：频繁出现"AI 返回内容不包含有效 JSON"或"JSON 括号不匹配"，用户需要重复输入。

**改进后**：四层回退提取策略 + 字符串感知括号匹配 + 自动重试（最多3次）+ 失败日志。

**技术亮点**：
- 括号匹配从简单计数升级为字符串感知（正确处理 JSON 字符串内的 `{}` 和转义引号）
- 新增 `<output>` 标签提取（prompt 层面引导模型输出到指定标签内）
- 重试时给模型更具体的 JSON 格式提醒
- 流式输出中过滤原始 JSON 显示（用户看到"AI 正在思考..."而非裸 JSON）

### 三、系统提示词：从"一个长 prompt"到"分层架构"

**改进前**：190+ 行密集指令，模型难以遵循。临界情况（模糊指令→JSON 输出失败）频繁报错。

**改进后**：借鉴 Claude Code 提示词设计原则，重构为 6 层结构：

| 层级 | 名称 | 解决的问题 |
|------|------|-----------|
| 第1层 | 身份锚定 | 明确角色边界，防止模型扮演无关角色 |
| 第2层 | 核心原则（6条） | 硬约束，用"最高优先级""违反将导致系统崩溃"等强调 |
| 第3层 | 决策框架（6级判断树） | 意图识别：何时执行 vs 何时询问，避免过度确认 |
| 第4层 | 输出格式 | `<output>` 标签包裹 JSON，代码层面精确提取 |
| 第5层 | 布局规则 | 思维导图固定坐标、流程图垂直排列、网格排列 |
| 第6层 | 自检清单 | 6 条输出前检查，降低 JSON 格式错误率 |

**关键设计决策**：
- 去掉了 `response_format: json_object`（与 DeepSeek v4 flash 流式模式不兼容）
- `max_tokens` 从 1024 提升到 2048（复杂思维导图需要更多 token）
- 思维导图示例从 3 分支增加到 5 分支，鼓励模型生成更丰富的内容

## 已知待解决问题

1. **思维导图/复杂图表的画布布局仍有错位**：LLM 的空间推理能力有限，仅靠 prompt 优化有天花板。可考虑：
   - 方案 A：前端侧实现自动布局算法（LLM 只输出节点和关系，坐标由前端计算）
   - 方案 B：引入专门的图布局库（如 dagre、elkjs）
2. **"AI 返回内容不包含有效 JSON"偶发**：虽有四层回退，但极端情况下仍可能失败。需持续观察日志优化
3. **TTS 音色依赖浏览器**：不同 OS/浏览器的音色质量差异大，未来可考虑接入云端 TTS（如 Azure Speech）
4. **无用户反馈闭环**：用户在画布上看到不满意的结果后无法自然表达，需要更好的"反馈→修正"流程
